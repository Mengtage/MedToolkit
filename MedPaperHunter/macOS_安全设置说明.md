# macOS 安全设置说明

## 问题：无法打开“启动系统.command”

当您双击运行“启动系统.command”时，可能会看到以下提示：

> “无法打开‘启动系统.command’，因为它来自身份不明的开发者。”

这是 macOS 的安全机制（Gatekeeper），用于保护您的 Mac 免受恶意软件的侵害。

---

## 解决方法 1：右键打开（推荐）

1. **右键点击**（或按住 Control 键点击）`启动系统.command` 文件
2. 选择 **“打开”**
3. 在弹出的确认对话框中，再次点击 **“打开”**

这样就可以正常运行了！

---

## 解决方法 2：通过系统偏好设置允许

1. 尝试双击运行 `启动系统.command`
2. 看到错误提示后，打开 **系统偏好设置**（System Preferences）
3. 点击 **安全性与隐私**（Security & Privacy）
4. 在 **通用**（General）标签页底部，您会看到：
   > “‘启动系统.command’已被阻止使用，因为它来自身份不明的开发者。”
5. 点击 **仍要打开**（仍要打开）
6. 再次尝试运行，在确认对话框中点击 **“打开”**

---

## 解决方法 3：使用终端运行（最简单）

如果以上方法太麻烦，可以直接在终端运行：

1. 打开 **终端**（Terminal）应用程序
2. 输入 `cd`（注意有空格），然后**将 MedPaperHunter 文件夹拖到终端窗口中**
3. 按 Enter 键
4. 输入以下命令并按 Enter：

```bash
python3 start_web_app.py
```

---

## 解决方法 4：移除隔离属性（进阶）

如果您想让文件以后可以直接双击运行，可以在终端中执行：

```bash
xattr -d com.apple.quarantine 启动系统.command
```

然后就可以直接双击打开了。

---

## 为什么会这样？

- 这个脚本是安全的，只是没有经过 Apple 签名
- 它只是一个简单的 shell 脚本，用于启动 Python 程序
- 您可以用文本编辑器打开查看它的内容，确认没有问题
