# 为什么不用大语言模型API也能智能检索？

## 🎯 核心答案

**因为我们的"智能"不是来自大语言模型，而是来自精心设计的规则和逻辑！**

---

## 🔍 我们是怎么做"智能检索"的？

### 1. 检索策略的多样性（就像你用多个关键词搜索一样）

就像你在Google搜索时，可能会尝试：
- 第一个搜索："人工智能 麻醉"
- 第二个搜索："机器学习 麻醉"
- 第三个搜索："AI 麻醉"

我们的代码就是**自动帮你完成这个过程！**

```python
# 我们生成5种不同的检索策略：
queries = [
    '严格检索（7天）',  # 关键词用AND，时间短
    '中等检索（30天）',
    '宽松检索（90天）',
    '关键词检索（365天）',
    'MeSH术语检索（365天）'
]
```

这就像你自己换着不同的关键词多次搜索一样！

---

### 2. 去重的逻辑（就像你检查是否重复一样）

当不同检索策略找到相同的文章时：
- 我们用PMID（PubMed唯一编号）来判断
- 就像你看文章标题是否重复一样

```python
if pmid not in all_articles:
    all_articles[pmid] = article  # 只保留新的
```

---

### 3. 关键词筛选（就像你快速读摘要判断是否相关）

不用大语言模型，我们也能判断文章是否相关！

**原理：**
- 从你的研究主题提取关键词：["embodied", "intelligence", "anesthesia"]
- 扩展同义词："ai" = ["artificial intelligence", "machine learning"]
- 检查文章标题和摘要中有没有这些词
- **标题中出现的词更重要！**（给更高权重）

```python
# 伪代码示例
if "embodied" in title:
    score += 3.0  # 标题匹配很重要
elif "embodied" in abstract:
    score += 1.0  # 摘要匹配次要

if score > 阈值:
    保留这篇文章！
```

这就像你快速读摘要，找关键词一样！

---

### 4. 检索式的构建（就像你在PubMed搜索框里写表达式）

你自己在PubMed搜索时，可能会写：
```
"artificial intelligence"[Title/Abstract] AND anesthesia[Title/Abstract]
```

我们的代码就是**自动帮你生成这个查询字符串！**

```python
# 代码做的事情
keywords = ["embodied", "intelligence", "anesthesia"]
mesh_terms = ["Cognition", "Intelligence", "Anesthesia"]

query = f"({' OR '.join(mesh_terms)}) AND ({' OR '.join(keywords[:3])})"
# 结果："(Cognition OR Intelligence OR Anesthesia) AND (embodied OR intelligence OR anesthesia)"
```

## 📊 总结：我们的"智能"是什么？

| 功能 | 怎么实现的 | 类比 |
|------|----------|------|
| **多个检索策略** | 换着不同关键词搜索 | 你在Google试多个查询 |
| **去重** | 用PMID做唯一标识 | 你检查是否重复文章 |
| **关键词筛选** | 检查标题摘要中有没有词 | 你快速读摘要找关键词 |
| **智能分段摘要** | 按比例切分 | 你凭直觉分段落 |
| **影响因子匹配** | CSV文件查找 | 你查期刊排名表 |

## 💡 类比：就像你自己做检索

假设你要找"人工智能麻醉"的文献：

**步骤1：你试多次搜索**
- "embodied intelligence anesthesia" → 找到3篇
- "AI anesthesia" → 找到5篇

**步骤2：你去重**
- 把找到的14篇放一起，去掉重复的，剩下10篇

**步骤3：你快速筛选**
- 读标题和摘要，只留下真正相关的，剩下7篇

**步骤4：你导出**
- 把这7篇整理成表格

---

## 🎉 为什么这样做更好？

| 对比 | 大语言模型API方式 | 我们的规则方式 |
|------|-----------------|-------------|
| **费用** | 每篇文章都要花钱 | 完全免费 |
| **网络要求** | 需要连接API服务器 | 只需要PubMed（免费） |
| **速度** | 慢（每篇文章要API请求） | 快（本地处理） |
| **稳定性** | 依赖API服务 | 100%稳定 |
| **隐私** | 文章内容要发给第三方 | 完全本地 |
| **透明度** | 不知道AI怎么想的 | 每一步规则都明确 |

---

## ✅ 我们什么时候真的用大语言模型？

只有在**你有API Key并主动开启AI筛选时**，我们才用大语言模型：
- AI筛选是**可选的**
- 是**锦上添花**的功能
- 不是必需的

---

总结：我们的"智能"是**来自精心设计的规则，不是来自大语言模型！** 🎉
