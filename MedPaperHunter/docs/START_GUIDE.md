# MedPaperHunter 快速开始指南

## 🎯 你的两个问题的答案

### 1. 删除 `automated_search_v2.py` 会失去什么功能？
**答案：不会失去任何功能！**
- `automated_search_v2.py` 只是一个简单的包装脚本
- 真正的核心功能都在 `advanced_search.py` 中
- 你可以直接运行 `python3 advanced_search.py` 来执行检索

### 2. 新的 Web 界面功能
现在你有了一个**漂亮的网页界面**，可以：
- ✅ 在网页上输入研究主题
- ✅ 选择检索策略（5种策略可选）
- ✅ 自定义时间范围（7天到5年）
- ✅ 实时查看检索进度
- ✅ 浏览结果列表
- ✅ 下载历史文件

## 🚀 快速启动（3种方式）

### 方式一：Web 界面（推荐）⭐️

```bash
# 1. 进入项目目录
cd MedPaperHunter

# 2. 启动 Web 应用
python3 app.py

# 3. 在浏览器中打开
# 访问：http://localhost:5000
```

### 方式二：直接运行高级检索

```bash
# 运行渐进式检索
python3 advanced_search.py
```

### 方式三：使用启动脚本（仅 Mac/Linux）

```bash
chmod +x run-web.sh
./run-web.sh
```

## 📁 你需要的核心文件

在你的 Mac 上，只需要保留这些文件：

```
MedPaperHunter/
├── app.py                    # ⭐ Web应用主文件
├── advanced_search.py         # ⭐ 核心检索引擎（必须）
├── requirements.txt           # 依赖列表
│
├── templates/                 # 网页模板
│   └── index.html            # 界面文件
│
├── data/                      # 数据目录
│   └── search_topic.txt      # 研究主题配置
│
└── output/                    # 输出目录（自动创建）
```

## 🎨 Web 界面功能介绍

### 1. 配置区域
- **研究主题**：输入你的研究方向
- **检索策略**：5种策略可选
  - 渐进式检索（推荐）：5个检索式，从严格到宽松
  - 严格检索：高精度，可能结果少
  - 中等检索：平衡精度和数量
  - 关键词检索：纯关键词匹配
  - MeSH术语检索：专业术语检索

- **时间范围**：7天 → 5年可选

### 2. 进度显示
- 实时显示检索进度
- 显示当前步骤
- 可视化进度条

### 3. 结果区域
- 显示找到的文献总数
- 列出前10条结果
- 每篇文献显示标题、作者、期刊、来源
- 可直接查看原文链接

### 4. 历史文件
- 列出所有历史检索结果
- 显示文件大小和时间
- 一键下载 CSV

## 📊 对比：命令行 vs Web界面

| 功能 | 命令行 | Web界面 |

| 输入主题 | 编辑文件或参数 | ✓ 网页输入 |
| 选择策略 | 修改代码 | ✓ 点击选择 |
| 时间范围 | 修改代码 | ✓ 下拉框选择 |

## ⚠️ 注意事项

1. **只需要 `advanced_search.py` 和 `app.py`**
   - 其他文件都是辅助的
   - 可以安全删除 `automated_search_v2.py`

2. **首次运行会自动创建目录**
   - `output/` 存放结果
   - `data/` 存放配置

3. **网络连接**
   - 需要访问 PubMed API
   - 如在中国，可能需要网络配置
